A Sprint 0

No universo do Desenvolvimento de Sistemas, a fase que antecede a primeira linha de código define o sucesso ou o fracasso de um produto digital. Em uma Sprint 0, equipes profissionais não abrem suas IDEs de forma aleatória; elas analisam os Requisitos Não Funcionais (RNF) e tomam decisões arquiteturais.

Hoje, vocês atuarão como squads de uma Software House, definindo as ferramentas para o MVP que será apresentado a investidores anjo na Feira de Inovação.

Fundamentação e Contextualização

A Ciência por Trás da Escolha da Stack.

Narrativa de TI

O Perigo do Hype

Imagine uma startup que decide construir seu aplicativo revolucionário usando a linguagem mais "da moda" no momento, sem analisar a viabilidade a longo prazo. Seis meses depois, o sistema precisa escalar, surgem bugs críticos e a empresa descobre que não há desenvolvedores qualificados no mercado para essa tecnologia específica.

Consequência

O banco de dados não suporta o tráfego e os investidores retiram o aporte financeiro. A empresa morre não por uma ideia ruim, mas por uma péssima decisão de Stack Tecnológica.

Conceituação Técnica

O que é a Stack?

A Stack Tecnológica é o conjunto de ferramentas, linguagens e frameworks que compõem as camadas de uma aplicação. No nosso ecossistema SENAI, trabalharemos com a tríade clássica e robusta:

PHP (Linguagem de Script Server-side)

Escolhida por sua maturidade, vasta documentação e compatibilidade nativa com quase todos os servidores web. No mercado, o PHP alimenta desde pequenos sites até gigantes como o Facebook e WordPress.

Conceituação Técnica

Banco de Dados e Client-side

MySQL (SGBD Relacional)

Um Sistema de Gerenciamento de Banco de Dados que utiliza a linguagem SQL. É o padrão de ouro para aplicações que exigem integridade de dados e relacionamentos complexos.

JavaScript (Client-side)

Essencial para a interatividade no navegador, permitindo que a aplicação responda ao usuário sem recarregar a página constantemente.

Conexão com a SA

Por que esta escolha?

A escolha desta stack para o seu projeto não é por acaso. Ela atende aos RNFs de interoperabilidade e baixo custo de infraestrutura.

Como vocês apresentarão para "investidores", precisam provar que a tecnologia escolhida é sustentável, segura e possui mão de obra disponível.

Ambiente Local

Ferramentas de Produtividade e o Ambiente Local.

Expansão Conceitual

Por que emular?

Diferente do HTML, o PHP não "roda" diretamente no navegador. Ele precisa de um interpretador no lado do servidor. Para não precisarmos de um servidor real na nuvem em cada aula, utilizamos o XAMPP.

Definição Técnica: XAMPP

O XAMPP é um pacote de software livre que consiste principalmente no servidor HTTP Apache, no banco de dados MariaDB (um fork do MySQL) e interpretadores para linguagens de script como PHP.

Analogia

Canteiro de Obras

Configurar o XAMPP é como montar um "canteiro de obras" isolado. Você testa tudo ali antes de levar para o prédio real (o servidor de produção).
Diagnóstico

Erros de Estudantes

  • Esquecer de iniciar os módulos: O erro mais comum é tentar acessar o projeto via localhost sem "dar Start" no Apache e MySQL no painel do XAMPP.
  • Caminho de arquivo incorreto: Salvar os arquivos fora da pasta C:\xampp\htdocs. Se o arquivo não estiver lá, o Apache não o encontrará.
  • Conflitos de porta: Outros programas (como Skype ou Docker) podem usar a porta 80, impedindo o Apache de iniciar.
Ferramentas

A Central de Comando

Um desenvolvedor sênior nunca trabalha sem uma IDE (Integrated Development Environment) potente. O VS Code será nossa central de comando. Ele não é apenas um editor de texto;

Funcionalidades

Com as extensões certas (PHP Intelephense, GitLens), ele se torna uma ferramenta de diagnóstico que avisa sobre erros de sintaxe antes mesmo de você rodar o código.

Versionamento e Estrutura

Versionamento com Git e Hierarquia de Projeto.

Colaboração

O Fluxo do Git

Trabalhar em equipe sem Git é um caos. O Git permite o Controle de Versão, garantindo que nenhum membro da equipe sobrescreva o código do outro.

1
Git Init
Inicializa o rastreamento.
2
Git Add/Commit
Cria um "ponto de salvamento" (checkpoint) com uma mensagem descritiva.
3
Git Push
Envia suas alterações para o repositório remoto (GitHub).
Organização

Hierarquia Profissional

Um projeto desorganizado é um projeto propenso a falhas. Adotaremos a seguinte hierarquia:

/frontend : Arquivos HTML, CSS e imagens. /backend : Scripts PHP, lógica de negócio e conexão com banco. /docs : Modelagem UML e Documento de Definição Tecnológica. .gitignore : Ignora arquivos desnecessários (config locais).
Justificativa

Decisão Técnica

As equipes devem agora preencher o Documento de Definição Tecnológica. Não basta dizer "vamos usar PHP". Vocês devem justificar:

"Usaremos PHP devido à facilidade de integração com o MySQL via PDO, garantindo a segurança contra SQL Injection exigida nos requisitos de segurança do projeto".

Parte 4

O Setup Final

Consolidação e Desafio.

Mercado

Boas Práticas

README.md impecável

O seu repositório no GitHub é o seu cartão de visitas. O arquivo README deve conter o título do projeto, a descrição do problema, as instruções de instalação e a lista de tecnologias.

Commits Semânticos

Evite mensagens como "ajuste" ou "teste". Use padrões como feat: adiciona login ou docs: atualiza manual.

Liderança

Início das Squads

O Líder Técnico de cada squad deve agora criar o repositório da organização, convidar os membros e garantir que todos consigam realizar o git clone e o primeiro git push de teste.

FAQ

Dúvidas Reais

"Professor, por que não usamos Node.js ou Python que são mais 'modernos'?"
PHP ainda domina cerca de 77% da web servida por linguagens server-side. Para um MVP em um semestre técnico, oferece a curva de aprendizado mais eficiente e ferramentas estáveis como o XAMPP.
"O XAMPP deu erro de porta 80 ocupada. O que eu faço?"
Acesse as configurações (Config) do Apache no painel, abra o arquivo httpd.conf e altere Listen 80 para Listen 8080. Acesse via localhost:8080.
FAQ

Git vs GitHub

FerramentaDefinição
GitSoftware instalado no PC para gerenciar versões.
GitHubPlataforma na nuvem para hospedar e colaborar em arquivos.
"Preciso commitar cada linha que eu escrevo?"
Não. Commite "unidades de trabalho completas". Ex: terminou a estrutura do cabeçalho ou a conexão com o banco. Commits pequenos e frequentes facilitam a correção.
Evidência de Desempenho

O Desafio

Objetivo: Configurar o ecossistema de desenvolvimento e o repositório oficial do projeto.

Requisitos Obrigatórios

  • Repositório no GitHub com convite para todos os membros e professores.
  • Pasta /docs com "Documento de Definição Tecnológica".
  • Arquivo .gitignore configurado.
