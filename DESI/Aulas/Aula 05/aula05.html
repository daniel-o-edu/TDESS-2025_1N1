Parte 1

A Anatomia do Documento: Doctype, Head e Body

Anatomia do Documento

A construção de uma página web assemelha-se à organização de um livro físico, mas com capacidades de interatividade e hipertexto que revolucionaram a comunicação humana. A transição das máquinas de escrever para a informação digital instantânea exigiu uma linguagem que desse estrutura ao caos da informação.

Anatomia do Documento

O <!DOCTYPE html>

Embora não seja uma "tag" HTML no sentido estrito de marcar conteúdo, o DOCTYPE é a primeira instrução que um navegador deve ler. Sua função é declarar o "modo de renderização". Em versões antigas (HTML 4.01), essa declaração era longa e complexa. No HTML5, ela foi simplificada para <!DOCTYPE html>. Sem ela, os navegadores podem entrar no "quirks mode" (modo de compatibilidade), o que faz com que o site seja exibido de forma inconsistente, ignorando padrões modernos de CSS.

Anatomia do Documento

A Estrutura Base: <html>

Uma página HTML padrão deve ser delimitada pelas tags <html> e </html>. Elas representam a raiz do documento. Dentro desta "casca" obrigatória, definimos duas seções fundamentais: o cabeçalho e o corpo.

Anatomia do Documento

O Cabeçalho: <head>

O <head> é a área declarativa da página. É um local para informações que, a princípio, não pretendemos mostrar diretamente no espaço de navegação. No entanto, seu papel é vital:

  • Metadados: Instruções para motores de busca e configurações de codificação (como o UTF-8 para garantir que acentos em português funcionem corretamente).
  • Título (<title>): É a única tag do cabeçalho que gera um efeito visível, mas não no corpo da página, e sim na borda da janela ou aba do navegador.
  • Vínculos externos: Onde conectamos arquivos de estilo CSS ou fontes externas que darão o "aspecto visual" aos elementos.
Anatomia do Documento

O Corpo: <body>

O <body> é a parte visível em si mesma. É aqui que o programador insere todo o conteúdo — textos, imagens, tabelas e formulários — que terá um efeito visível para o usuário final. A ordem das tags é rígida: o <head> deve preceder o <body>, e ambos devem estar contidos em <html>.

Fixação

Exercício de Fixação 1.1

Crie a estrutura básica de um arquivo chamado index.html. No <head>, defina o título como "Minha Jornada HTML5". No <body>, escreva um parágrafo (<p>) explicando por que a separação entre o que é "declarativo" e o que é "visível" é importante para a organização do código.

Modo Detetive: Inspecionando a Web com DevTools

Modo Detetive

Para entender como a Internet e as páginas Web funcionam, não basta apenas ler o código; é preciso observar como os navegadores interpretam e executam esses arquivos em tempo real. As Ferramentas do Desenvolvedor (DevTools) são o "microscópio" do programador moderno.

Modo Detetive

O Navegador como Interpretador

Um navegador web é um programa projetado para ler e interpretar arquivos acessados pela Internet. Quando abrimos o DevTools (geralmente com a tecla F12), estamos acessando a camada de processamento do navegador.

Modo Detetive

A Aba Elements (Elementos)

Nesta aba, vemos o DOM (Document Object Model). O DOM é a representação viva do seu HTML. Diferente do seu arquivo de texto estático editado no bloco de notas, o DOM reflete todas as alterações feitas via scripts ou erros de interpretação do navegador.

  • Inspecionar: Ao clicar com o botão direito em qualquer parte de um site e selecionar "Inspecionar", o DevTools destaca exatamente onde aquele elemento está no código.
  • Edição em tempo real: Você pode alterar textos, deletar imagens ou mudar cores diretamente no navegador. Isso é útil para prototipagem rápida, embora as mudanças desapareçam ao atualizar a página (pois o arquivo original em disco permanece intacto ).
Modo Detetive

Diagnosticando Erros

Diferentes navegadores têm tolerâncias distintas aos erros. Se você esquecer de fechar uma tag ou escrever um comando mal escrito, o navegador tentará "adivinhar" o que você quis fazer. O DevTools mostra essa "interpretação" do navegador, permitindo que o desenvolvedor identifique falhas de hierarquia que não são óbvias apenas olhando para o código fonte.

Fixação

Atividade de Fixação 2.1

Acesse o site de um grande portal de notícias. Use o DevTools para encontrar:

  1. A tag <h1> principal.
  2. Quantas tags <img> existem na página inicial.
  3. Tente alterar o título de uma notícia para "Aprendendo DevTools com Limongi". O que acontece se você atualizar a página? Explique o porquê com base no conceito de "arquivo em disco" versus "interpretação do navegador".

Arquitetura Semântica: Do <div> ao Significado Real

Arquitetura Semântica

Na era inicial do HTML, programadores utilizavam tags como <center> para centralizar textos ou <font> para mudar cores. No entanto, a evolução da web exigiu que as tags não apenas dissessem "como algo deve parecer", mas sim "o que algo é". Isso é o que chamamos de Semântica.

Arquitetura Semântica

A Limitação das Divs

A tag <div> como uma das melhores formas de organizar a informação. Por muito tempo, as <div>s (divisores) foram usadas para tudo: cabeçalhos, menus, rodapés. O problema é que, para um motor de busca como o Google ou um leitor de tela para cegos, uma <div> não tem significado; ela é apenas uma "caixa genérica".

Arquitetura Semântica

As Novas Tags de Estrutura

O HTML5 introduziu tags que dão significado à estrutura que antes era feita apenas com <div>s com atributos de ID ou classe:

  • <header>: Define o cabeçalho de uma página ou de uma seção específica. Contém logos, títulos e, frequentemente, a navegação.
  • <nav>: Reserva-se especificamente para blocos de links de navegação principal. Não deve ser usada para qualquer link, mas para os caminhos essenciais do site.
  • <main>: Representa o conteúdo principal e exclusivo daquele documento. Deve haver apenas um por página.
Arquitetura Semântica

As Novas Tags de Estrutura (Cont.)

  • <section>: Agrupa conteúdos relacionados tematicamente. Pense nela como os "capítulos" mencionados por Limongi.
  • <footer>: O rodapé. Contém informações de copyright, contatos ou links secundários.
Arquitetura Semântica

Benefícios: Acessibilidade e SEO

Utilizar tags semânticas é fundamental para a acessibilidade. Leitores de tela utilizam essas marcas para permitir que usuários com deficiência visual saltem diretamente para o "conteúdo principal" (<main>) ou encontrem o "menu de navegação" (<nav>) sem ter que ler a página inteira. Além disso, melhora o SEO (Search Engine Optimization), pois ajuda os robôs de busca a entenderem a hierarquia da informação.

Fixação

Atividade de Fixação 3.1

Desenhe o esqueleto de um site de receitas. Utilize as tags semânticas aprendidas para organizar:

  • O nome do site e o menu.
  • A área central com o título da receita (<h1>) e o modo de preparo.
  • Uma área lateral com ingredientes.
  • O rodapé com o nome do chef.

Escreva o código HTML correspondente para essa estrutura.

Atividade Diagnóstica

O Caso Clínico

Neste desafio final, vocês atuarão como consultores de desenvolvimento web para corrigir um "paciente" (um site) que sofre de "Divite Aguda" — o uso excessivo de tags genéricas sem significado.

Analise o código abaixo. Ele funciona visualmente, mas é um desastre em termos de boas práticas modernas e acessibilidade.

Atividade Diagnóstica

Missão

Refatore o código acima seguindo estas diretrizes:

  • 1. Adicione a Declaração de Tipo: Insira o DOCTYPE correto.
  • 2. Substitua as Divs: Troque as <div>s com IDs por tags semânticas adequadas (<header>, <nav>, <main>, <article> ou <section>, e <footer>).
  • 3. Melhore a Hierarquia: Certifique-se de que os títulos (<h1>, <h2>) estão seguindo uma ordem lógica de importância.
Relatório Técnico

Justificativa Técnica (Mínimo 300 palavras)

Após realizar a correção, escreva um pequeno relatório explicando:

  • Por que a tag <header> é melhor que <div id="topo"> para um leitor de tela.
  • Qual a diferença entre usar <main> e apenas colocar o conteúdo solto no <body>.
  • Como a estrutura correta de títulos (h1 a h6) ajuda os motores de busca a indexar seu site.
